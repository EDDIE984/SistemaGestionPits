Webservices PDA

Consumo de servicios para operarios de isla

Esta pagina documenta los endpoints creados para autenticar operarios y registrar inicio, pausa, reanudacion y finalizacion de trabajos desde una app Flutter ejecutada en PDA con lector de codigo de barras.

Base URL en Vercel

Reemplaza el dominio por el de tu despliegue.

GET /api/pda-health POST /api/logeo_Operaciones POST /api/operacion-isla

Funcionamiento en Vercel

Si la app esta desplegada en Vercel, los archivos dentro de api/ se ejecutan como Serverless Functions. Por eso los servicios quedan disponibles bajo el mismo dominio de la aplicacion: 

https://tu-dominio.vercel.app/api/logeo_Operaciones
https://tu-dominio.vercel.app/api/operacion-isla

En el despliegue actual de PITS, la base URL es: 

https://sistema-gestion-pits.vercel.app
Variables requeridas en Vercel: SUPABASE_SERVICE_ROLE_KEY y SUPABASE_URL. Si no existe SUPABASE_URL, el servicio usa VITE_SUPABASE_URL como respaldo.
Importante: la llave SUPABASE_SERVICE_ROLE_KEY nunca debe enviarse al frontend ni a Flutter. Solo vive en Environment Variables de Vercel.

0. Verificacion del servicio

GET /api/pda-health

Permite validar que las funciones PDA estan publicadas y que Vercel tiene configuradas las variables necesarias para conectarse a Supabase. No expone valores secretos, solo confirma si existen.

cURL

curl "https://sistema-gestion-pits.vercel.app/api/pda-health"

Respuesta esperada

{
  "ok": true,
  "service": "pda-webservices",
  "env": {
    "hasSupabaseUrl": true,
    "hasServiceRoleKey": true
  }
}
Si este endpoint responde 404, el deploy activo no contiene el archivo api/pda-health.ts. Si responde 500 con ok: false, falta configurar alguna variable de entorno en Vercel.

1. Login operaciones

POST /api/logeo_Operaciones

Valida las credenciales contra la funcion validate_credentials, confirma que el usuario tenga perfil OPERARIO y devuelve sucursal, perfil, isla y tecnico_id.

Request

Campo Tipo Requerido Detalle
username o usuario string Si Usuario del operario.
password o clave string Si Clave del operario. 
{
  "username": "PMORA",
  "password": "tu_clave"
}

Respuesta exitosa

{
  "ok": true,
  "usuario": {
    "id": "uuid-usuario",
    "nombre": "Nombre del operario",
    "username": "PMORA",
    "perfil": "OPERARIO",
    "sucursal": {
      "id": "uuid-sucursal",
      "nombre": "Carondelet"
    },
    "isla": {
      "id": "uuid-isla",
      "nombre": "Pintura"
    },
    "tecnico_id": "uuid-tecnico"
  }
}

cURL

curl -X POST "https://sistema-gestion-pits.vercel.app/api/logeo_Operaciones" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "PMORA",
    "password": "tu_clave"
  }'

Postman

Method POST
URL https://sistema-gestion-pits.vercel.app/api/logeo_Operaciones
Headers Content-Type: application/json
Body raw y formato JSON
Si la respuesta es {"ok":false,"error":"Metodo no permitido"}, la solicitud no llego como POST. Revisa el selector de metodo en Postman, que la URL sea /api/logeo_Operaciones y que no estes probando /api/pda-health.

2. Operacion de isla

POST /api/operacion-isla

Registra la operacion enviada desde el PDA. Actualiza la tarea en orden_isla_tareas, inserta bitacora en orden_isla_tarea_eventos y registra pausas en orden_isla_tarea_pausas.

Acciones soportadas

INICIAR
PAUSAR
REANUDAR
FINALIZAR

Request

Campo Tipo Requerido Detalle
usuario_id uuid Si ID devuelto por el login.
numero_orden o ot string Si, salvo que envies tarea_id Numero de OT leido desde codigo de barras, por ejemplo OT-2026-001.
accion o operacion string Si INICIAR, PAUSAR, REANUDAR o FINALIZAR.
tarea_id uuid No Util si el codigo de barras identifica una tarea especifica.
tecnico_id uuid No Se puede omitir; el servicio lo busca por el usuario_id.
observacion string No Texto libre para la bitacora del evento.
motivo_pausa string No Motivo guardado cuando la accion es PAUSAR.

Iniciar trabajo

{
  "usuario_id": "uuid-usuario",
  "numero_orden": "OT-2026-001",
  "accion": "INICIAR",
  "observacion": "Inicio desde PDA"
}

{
  "usuario_id": "uuid-usuario",

Flujo recomendado para Flutter

1. Login

La app llama a /api/logeo_Operaciones. Guarda en memoria local segura el usuario.id, tecnico_id, sucursal.id e isla.id.

2. Lectura de codigo

El PDA lee el codigo de barras de la OT. Si el codigo solo trae numero de OT, envia numero_orden. Si trae tarea, envia tarea_id.

3. Accion operativa

Segun el boton del operario, envia INICIAR, PAUSAR, REANUDAR o FINALIZAR.

4. Actualizar pantalla

Usa la respuesta para mostrar estado de tarea, estado de OT y hora del evento.

Errores HTTP

Codigo Significado Ejemplo
400 Faltan campos o accion invalida. usuario_id es requerido
401 Credenciales invalidas o usuario inactivo. Credenciales invalidas
403 Usuario no operario, otra sucursal o tarea no asignada. La OT no pertenece a la sucursal del operario
404 No se encontro OT o tarea. OT no encontrada
405 Metodo HTTP incorrecto para el endpoint. Metodo no permitido
409 Transicion no permitida por estado actual. Solo se puede pausar una tarea en proceso
500 Error interno o configuracion faltante. Falta configurar SUPABASE_SERVICE_ROLE_KEY

Ejemplo Flutter

import 'dart:convert';
import 'package:http/http.dart' as http;

const baseUrl = 'https://tu-dominio.vercel.app';

Future<Map<String, dynamic>> loginOperaciones(
  String username,
  String password,
) async {
  final res = await http.post(
    Uri.parse('$baseUrl/api/logeo_Operaciones'),
    headers: {'Content-Type': 'application/json'},
    body: jsonEncode({
      'username': username,
      'password': password,
    }),
  );

  final data = jsonDecode(res.body) as Map<String, dynamic>;
  if (res.statusCode != 200 || data['ok'] != true) {
    throw Exception(data['error'] ?? 'Error de login');
  }
  return data;
}

Future<Map<String, dynamic>> registrarOperacion({
  required String usuarioId,
  required String numeroOrden,
  required String accion,
  String? observacion,
}) async {
  final res = await http.post(
    Uri.parse('$baseUrl/api/operacion-isla'),
    headers: {'Content-Type': 'application/json'},
    body: jsonEncode({
      'usuario_id': usuarioId,
      'numero_orden': numeroOrden,
      'accion': accion,
      if (observacion != null) 'observacion': observacion,
    }),
  );

  final data = jsonDecode(res.body) as Map<String, dynamic>;
  if (res.statusCode != 200 || data['ok'] != true) {
    throw Exception(data['error'] ?? 'Error registrando operacion');
  }
  return data;
}
